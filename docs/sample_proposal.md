# Sales Meeting Notes Automation Proposal

## My Understanding

The request is not to build a heavy system first. The immediate need is to take 6-10 sales meeting notes per day from mixed sources and normalize them into a consistent structure, then extract:

- customer pain points
- next actions
- owner

The desired outputs are:

- one daily Markdown summary
- one structured CSV export

## Suggested Deliverables

- a runnable local tool for batch processing meeting notes
- support for docx, txt/md, and pasted text in the first version
- one normalized format per note
- one combined Markdown summary
- one CSV export
- a short usage guide

## Delivery Steps

1. Confirm sample inputs and output format
2. Define extraction rules and output template
3. Build the first runnable version
4. Test with real samples
5. Deliver the tool and guide

## Timeline

- Day 1: confirm samples and target fields
- Day 2: implement the processing flow and export
- Day 3: test and deliver a usable first version
