# 客户文件批量重命名与订单号提取本地脚本提案

## 我对需求的理解

您当前每周需要处理约 300 份客户文件，主要有两件事：

1. 按固定规则批量重命名文件。
2. 从文件名或文件内容中提取订单号，整理成一份 CSV 表格。

已知约束是全程本地处理，不能上传云端；交付物希望是一个本地可运行脚本，加上一份 CSV 导出结果；您可以提供一批样本文件和一份包含预期订单号的对照表。

目前还有几个信息缺口，但不影响先做初版方案：

- 文件重命名规则还不明确：是按订单号、客户名、日期，还是其他字段命名。
- 订单号来源还不明确：来自文件名、正文内容、PDF/图片识别，还是 Excel/Word 元数据。
- 文件类型还不明确：PDF、Excel、Word、图片、压缩包，还是混合类型。
- CSV 需要哪些字段还不明确：例如原文件名、新文件名、订单号、处理结果、失败原因。
- 您的本地运行环境还不明确：Windows 还是 macOS，是否可安装 Python。

## 建议交付内容

- 一个本地批处理脚本。
- 一份使用说明文档，包含运行步骤、输入输出目录说明、异常处理说明。
- 一份 CSV 导出文件。
- 一轮基于您样本数据的规则适配与验证。
- 可选增加日志文件，便于后续核对哪些文件成功、哪些失败。

建议默认采用 Python 本地脚本实现，原因是兼容性高、后续规则调整成本低、适合每周重复使用。如果您指定必须用 Excel VBA 或 PowerShell，也可以按环境调整。

## 实施步骤

1. 接收您提供的样本文件和预期订单号对照表。
2. 先确认重命名规则、订单号提取规则、CSV 字段结构。
3. 基于样本做首版脚本，先跑一小批文件验证结果。
4. 调整异常规则，例如重名文件、识别失败、缺少订单号、格式不一致。
5. 在完整样本上执行并导出 CSV。
6. 交付脚本、说明文档和本次导出的 CSV。
7. 预留一次小幅规则微调，确保后续每周可直接复用。

## 时间预估

- 需求确认：0.5 天
- 首版开发与样本验证：0.5 至 1 天
- 调整与最终交付：0.5 天

在样本资料齐全、规则明确的前提下，标准交付可控制在 1 至 2 天内，满足 3 天内交付要求。

## 风险与边界

- 如果订单号不在文件名里，而是在扫描件图片或复杂 PDF 正文里，识别难度和准确率会明显上升，需单独评估。
- 如果样本文件格式混杂较多，可能需要分类型分别处理。
- 本次报价默认不包含长期驻场维护，不包含云端系统接入，不包含数据库或后台管理页面。
- 如果后续命名规则频繁变化，建议将规则做成可配置项，这部分可在标准版基础上扩展。
- 若客户本地环境无法运行 Python，需要改成交付可执行文件或指定环境脚本，可能会增加少量适配时间。
