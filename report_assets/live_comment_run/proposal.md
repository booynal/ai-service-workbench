# 文件重命名与订单号提取自动化脚本提案

## 我对需求的理解

您当前每周需要处理约 300 个客户文件，核心工作有两部分：

1. 按固定规则批量重命名文件。
2. 从文件名或文件内容中提取订单号，整理成一份 CSV 表格。

本次需求的目标不是做在线系统，而是交付一套可在本地电脑直接运行的脚本，外加一份 CSV 导出结果，且全程本地处理，不上传云端，满足数据合规要求。

目前可以直接启动，但还有几个关键信息缺口需要在开始前确认：

- 现有文件名规则：现在的命名格式是否统一，是否有多个来源格式。
- 目标命名规则：重命名后的格式需要明确，例如“客户名_订单号_日期.pdf”。
- 订单号来源：订单号是在文件名里、文件内容里，还是两者都有可能。
- 文件类型：样本文件是 PDF、Excel、图片、Word，还是混合格式。
- 运行环境：脚本需要在 Windows 还是 macOS 本地执行。

在这些信息未完全补齐的情况下，仍然可以先按“样本驱动”方式做第一版，先适配您提供的样本文件夹和对照表，再根据测试结果微调规则。

## 建议交付内容

建议交付以下内容：

1. 本地批处理脚本 1 套  
   用于批量扫描指定文件夹、按规则重命名文件、提取订单号并生成 CSV。

2. CSV 导出文件 1 份  
   至少包含：
   - 原文件名
   - 新文件名
   - 提取到的订单号
   - 处理结果
   - 异常备注

3. 使用说明文档 1 份  
   包含脚本运行方式、输入目录要求、输出结果说明、常见报错处理。

4. 样本验证结果 1 轮  
   用您提供的样本文件和订单号对照表进行测试，确保规则可落地。

如您后续有需要，也可以预留扩展位：

- 处理失败文件单独输出
- 重命名前先生成预览清单
- 支持多种文件命名规则
- 支持二次复核日志

## 实施步骤

1. 确认规则  
   基于样本文件夹和订单号表，确认文件命名模式、订单号识别方式、最终重命名格式。

2. 制作第一版脚本  
   完成本地扫描、批量重命名、订单号提取、CSV 导出四个核心流程。

3. 样本测试  
   用您提供的样本跑一轮，核对订单号识别准确率和文件重命名结果。

4. 调整异常规则  
   对格式不一致、重复文件名、未识别订单号等情况补充处理逻辑。

5. 交付正式版本  
   交付脚本、CSV 样例、使用说明，并说明后续如何自行重复使用。

## 时间预估

在资料齐全的前提下，建议按以下节奏执行：

- 第 1 天：确认规则、分析样本、确定输出字段
- 第 2 天：完成脚本开发与首轮测试
- 第 3 天：修正规则、补充异常处理、正式交付

如果样本格式比较统一，通常 2 天内可以完成标准版交付。  
如果样本文件来源较杂、命名规则不一致较多，建议按 3 天窗口执行更稳妥。

## 风险与边界

1. 如果样本文件格式差异过大，第一版只能覆盖主流规则，长尾异常需要二次补充。
2. 如果订单号存在于扫描件图片中，而非文件名或可读取文本内，则可能需要 OCR，这属于额外工作范围。
3. 如果目标重命名规则后续频繁变更，会影响脚本逻辑，需要按变更量评估增补费用。
4. 本次默认交付“本地脚本 + CSV”，不包含桌面界面、小程序、网页后台。
5. 默认仅针对您提供的样本类型适配；新增文件类型需要单独确认。
6. 如果客户电脑缺少运行环境，可能需要补充一次本地部署指导。
