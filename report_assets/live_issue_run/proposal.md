# 本地文件批量重命名与订单号提取脚本提案

## 我对需求的理解

您当前每周需要处理约 300 份客户文件，核心有两件事：

1. 按固定规则批量重命名文件，减少人工逐个修改。
2. 从这些文件中提取订单号，整理成一份可直接打开使用的 CSV 表格。

已知约束是必须本地处理，不能上传云端；您目前能提供一份样例文件夹，以及一份包含预期订单号的对照表。交付目标是一个本地可运行脚本，加一份 CSV 导出结果。

目前还有几个关键信息未明确，但不影响先做初版方案：
- 文件类型是什么：`pdf / xlsx / docx / 图片 / 混合`
- 订单号是从文件名提取，还是从文件内容提取
- 重命名规则是否固定：例如“客户名 + 订单号 + 日期”
- 是否存在重名、缺失订单号、异常文件
- 客户本地运行环境：`Windows`、`macOS` 还是两者都要支持

在这些信息未补齐前，初版可按“先基于样例建立规则，优先支持主流文件类型和固定命名规则”的方式落地。

## 建议交付内容

1. 一个本地批处理脚本  
   默认建议用 `Python` 实现，方便本地运行、后续维护和扩展。

2. 一份 CSV 导出文件  
   至少包含：
   - 原文件名
   - 新文件名
   - 提取出的订单号
   - 处理结果
   - 异常原因（如未识别）

3. 一份简短使用说明  
   包括：
   - 如何放置待处理文件
   - 如何运行脚本
   - 输出文件在哪里
   - 异常文件如何查看

4. 可选的异常处理机制  
   例如：
   - 未识别订单号的文件单独标记
   - 重名文件自动跳过或追加序号
   - 输出处理日志，方便复核

## 实施步骤

1. 分析您提供的样例文件和订单号对照表，确认订单号提取规则与重命名规则。
2. 先做一版规则验证脚本，在样例数据上跑通，并核对提取准确率。
3. 完成正式脚本，支持批量重命名和 CSV 导出。
4. 增加基础异常处理，避免误改文件或覆盖重名文件。
5. 用您提供的样例做一次完整测试，交付脚本、CSV 示例和使用说明。

## 时间预估

在样例资料完整、规则能明确的前提下：

- 需求确认与样例分析：0.5 天
- 脚本开发与规则调试：1 天
- 测试、修正、交付说明：0.5 天

正常可在 `2 天内` 完成交付，符合您 `3 天内` 的要求。  
如果样例文件类型复杂，或订单号需要从文件内容而不是文件名中识别，时间可能增加 `0.5-1 天`。

## 风险与边界

1. 如果订单号提取依赖文件内容识别，难度会明显高于从文件名提取，报价和工期可能上调。
2. 如果样例文件本身命名不统一、质量差、规则有大量例外，需要先定义“主规则 + 例外处理方式”。
3. 本次方案默认是本地脚本，不含云端、网页后台、数据库系统。
4. 本次默认交付“可运行工具”，不包含长期运维服务。
5. 若客户后续希望增加图形界面、拖拽操作、多目录批处理、更多字段提取，可作为二期扩展。
