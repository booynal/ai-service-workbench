# 本地文件批量重命名与订单号提取脚本提案

## 我对需求的理解

您现在每周需要处理约 300 个客户文件，重复性工作主要有两部分：

1. 按固定规则批量重命名文件。
2. 从文件名或文件内容中提取订单号，整理成一份 CSV 表格。

结合您给出的约束，我理解本项目的目标是：

- 提供一个可在本地电脑直接运行的脚本，不依赖云服务。
- 输入为您现有的文件夹样本和订单号对照表。
- 输出为：
  - 一套可复用的本地批处理脚本
  - 一份按要求导出的 CSV 结果文件
- 交付周期控制在 3 天内。

当前还存在几个信息缺口，但不影响先出初版方案：

- 文件重命名规则是否已经固定，格式示例是什么。
- 订单号是从文件名提取，还是需要读取文件内容。
- 文件类型有哪些，例如 PDF、Excel、图片、Word。
- 使用环境是 Windows 还是 macOS。
- 遇到重名文件、缺失订单号、异常格式时，是否需要生成错误清单。

## 建议交付内容

建议本次交付包含以下内容：

- 本地自动化脚本 1 套
  - 批量扫描指定文件夹
  - 按规则重命名文件
  - 提取订单号
  - 导出 CSV
- 使用说明文档 1 份
  - 如何放置源文件
  - 如何运行脚本
  - 输出文件在哪里查看
- CSV 示例结果 1 份
- 异常处理机制
  - 无法识别的文件单独记录
  - 重名冲突自动跳过或追加后缀
- 可选：简单配置项
  - 输入目录
  - 输出目录
  - CSV 字段名
  - 重命名模板

建议技术实现优先采用 Python，本地运行方便，后续维护成本低。

## 实施步骤

1. 先基于您提供的样本文件和订单号表，确认命名规则与订单号识别规则。
2. 做一版本地脚本原型，先跑通 20 到 50 个样本文件。
3. 根据样本结果修正边界情况，例如重名、空值、格式不一致。
4. 完成正式版脚本，加入 CSV 导出和异常记录。
5. 用您提供的完整样本做一次本地验证，交付脚本、说明和示例结果。

## 时间预估

- 需求确认与样本分析：0.5 天
- 脚本开发与初版联调：1 天
- 异常处理、优化与交付说明：0.5 到 1 天

总计预计：2 到 3 天内可交付。

## 风险与边界

- 如果文件命名规则不统一，脚本识别准确率会受样本质量影响。
- 如果订单号需要从 PDF、图片扫描件中读取，工作量会明显高于“从文件名提取”。
- 若样本文件覆盖不全，正式运行时可能出现新增异常格式，需要二次补规则。
- 本次报价默认是本地单机脚本，不包含：
  - 云端部署
  - 多人协同系统
  - 图形界面软件
  - 长期运维服务
- 如需兼容多种文件类型或加入可视化界面，建议单独拆分二期。
