# 本地文件批量重命名 + 订单号提取脚本提案

1. 每周要处理约 300 个客户文件，手工重命名重复且容易出错。
2. 还需要从这些文件里提取订单号，并整理成一份表格。

基于你提供的信息，我理解本项目的目标是：  
开发一个**纯本地运行**的自动化脚本，不依赖云端，把指定文件夹中的文件按规则批量重命名，同时提取订单号并导出一份 CSV。

- 文件类型未明确：PDF、Excel、Word、图片、压缩包，还是混合格式。
- CSV 需要哪些列未明确：例如原文件名、新文件名、订单号、处理状态、备注。
- 本地运行环境未明确：Windows 还是 macOS，是否可以安装 Python。

## 建议交付内容

建议交付以下内容：

1. 一个本地批处理脚本  
   支持读取指定文件夹，批量处理文件，不上传任何数据。

2. 文件重命名功能  
   按确认后的命名规则自动重命名，并保留处理日志，避免重名覆盖。

3. 订单号提取功能  
   从文件名或文件内容中提取订单号，并与样例表进行校验。

4. 一份 CSV 导出结果  
   默认建议包含以下字段：`原文件名`、`新文件名`、`订单号`、`处理结果`、`异常说明`

5. 使用说明文档  
   包括运行方式、目录结构要求、常见报错处理。

如样例足够稳定，优先建议用 **Python 本地脚本** 实现，后续维护成本低，也方便你继续重复使用。
