# 多来源资料整理成结构化报告提案

## 我对需求的理解

您当前的问题不是“没有资料”，而是资料来源太散：网页、会议纪要、聊天记录、文档、截图都有，导致后续无法快速看结论，也难以继续分发给同事执行。

本次项目的目标，是把这些原始资料整理成一份结构化报告，输出清晰的结论、关键事实、行动建议和待确认项，让您可以直接转发、汇报或继续执行。

## 建议交付内容

1. 一份结构化主报告  
   默认用 Markdown 或 PDF 交付，方便二次编辑和归档。

2. 一份关键信息摘要  
   提炼关键结论、重点风险、下一步动作。

3. 可选附录  
   包含原始信息索引、来源映射、待确认问题列表。

4. 如果资料量较大，可附带一份 CSV 或表格型索引，方便您后续检索。

## 实施步骤

1. 接收原始资料并统一整理格式。
2. 按主题或时间线进行信息归类。
3. 提炼结论、关键事实和建议。
4. 生成初版报告并做一轮结构修正。
5. 输出正式版报告。

## 时间预估

- 资料较少、结构明确：0.5 到 1 天
- 资料中等规模：1 到 2 天
- 来源多、格式杂：2 到 3 天

## 风险与边界

- 如果原始资料缺上下文，部分结论只能标记为待确认。
- 如果截图内容很多且文字质量差，信息抽取时间会增加。
- 默认不包含公开发表版文案润色，如需对外稿件可单独扩展。

如果你现在主要问题是“资料很多但很难迅速看出结论”，这类交付会比继续自己整理省时间很多。
