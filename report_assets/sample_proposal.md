# 销售会议纪要整理自动化提案

## 我对需求的理解

你的核心需求不是做一个复杂系统，而是先把每天 `6-10` 份来源不统一的销售会议纪要，稳定整理成统一格式，并自动提取以下关键信息：

- 客户痛点
- 下一步行动项
- 负责人

最终希望每天自动产出：

- `1` 份汇总版 `Markdown`
- `1` 份结构化 `CSV`

当前输入来源比较杂，包括：

- Word 文档
- 飞书导出文本
- 手工复制文本

另外你提到预算有限，但如果首版效果稳定，后续有持续使用的可能，所以这次更适合先做一个 `3 天内可跑的 MVP`，优先解决“能稳定处理、输出统一、方便复核”，而不是一开始就做重系统或复杂界面。

## 建议交付内容

建议首版交付以下内容：

- 一个可运行的纪要整理工具，支持批量处理当天会议纪要
- 支持首版输入格式：
  - `docx`
  - `txt / md`
  - 手工粘贴文本
- 自动输出统一格式的单篇纪要结构
- 自动汇总生成当日总览 `Markdown`
- 同步导出 `CSV`，便于后续筛选、统计或导入其他系统
- 一份简短使用说明，确保后续可自行重复使用

建议首版 `CSV` 字段包含：

- 日期
- 客户名称
- 纪要来源
- 客户痛点
- 下一步行动项
- 负责人
- 备注 / 待确认项

首版默认不做复杂后台，优先做“本地可跑”或“固定目录批量处理”的轻量方案，性价比更高。

## 实施步骤

1. 确认输入样本与目标格式  
   你提供 `3-5` 份真实纪要样本，我这边据此统一字段结构和输出模板。

2. 定义提取规则与输出模板  
   先明确 Markdown 模板、CSV 字段、负责人识别规则、行动项拆分规则。

3. 开发首版可运行工具  
   实现批量读取、文本清洗、统一格式整理、信息提取、Markdown/CSV 导出。

4. 用真实样本调试  
   用你提供的真实数据跑一轮，修正字段遗漏、格式不齐、提取偏差等问题。

5. 交付与说明  
   交付可运行版本、样例输出、使用说明，并保留一轮小调整。

## 时间预估

在样本到位、需求边界确认及时的前提下：

- 第 1 天：拿样本、定模板、确定字段和边界
- 第 2 天：完成首版处理流程和导出功能
- 第 3 天：用真实样本调试，修正问题，交付可跑版本

如果中间出现以下情况，时间会顺延：

- 输入格式差异过大
- 样本质量不稳定
- 输出字段频繁变更
- 需要兼容老 `.doc`、图片扫描件、复杂表格内容

## 报价建议

### 基础版

`￥3000 - ￥4000`

适合预算敏感、先验证可行性的情况。

包含：

- 支持 `txt/md` + 手工粘贴文本
- 输出统一 `Markdown`
- 导出基础 `CSV`
- `1` 轮小调整
- 交付周期约 `4-5` 个工作日

不包含：

- 复杂 Word 兼容
- 规则深度优化
- 加急排期

### 标准版

`￥5000 - ￥7000`

适合你当前这个需求，建议优先选这一档。

包含：

- 支持 `docx`、飞书导出文本、手工复制文本
- 输出单篇统一格式 + 每日汇总 `Markdown`
- 导出结构化 `CSV`
- `2` 轮基于真实样本的提取优化
- 简短使用说明
- 交付周期约 `3` 个工作日

### 加急版

`￥8000 - ￥10000`

适合明确要求 `3 天内必须落地`，且希望沟通和修正响应更快的情况。

包含：

- 标准版全部内容
- 优先排期
- 每日同步进度
- 更快的问题修正响应
- 可压缩到 `2-3` 天内交付首版

## 风险与边界

目前信息还缺以下几项，但不影响先给出初版方案：

- 缺少真实样本，暂时无法判断纪要噪声程度
- 缺少最终 `Markdown` 模板
- 缺少 `CSV` 字段的最终定义
- 不清楚运行环境是本地电脑还是服务器
- 不清楚是否允许接入外部模型/API
- 不清楚是否需要兼容老 `.doc`、扫描件或图片 OCR

首版边界建议先明确如下：

- 首版优先保证“能跑通 + 输出统一 + 可人工复核”，不承诺一次性做到 `100%` 准确提取
- 老 `.doc`、图片、扫描 PDF、复杂表格内容，建议放到二期
- 如果不同纪要里客户名称、负责人写法非常不统一，首版可能需要人工补一遍
- 如果后续需要接飞书 API、云端部署、权限管理、界面化操作，建议单独作为下一阶段报价
