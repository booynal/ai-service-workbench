# 快速报价口径

## 什么时候报基础版

满足大部分以下条件时：

1. 输入格式单一
2. 规则比较清晰
3. 例外情况少
4. 对方更在意先便宜试一下

典型口径：

```text
这个需求可以先按基础版做，先把最核心的流程跑通，适合先验证可行性。
```

## 什么时候报标准版

满足任意几项时优先报标准版：

1. 后续会重复使用
2. 需要日志或异常处理
3. 输入数据有一定脏乱差
4. 不能只做一次性演示

典型口径：

```text
如果你是准备后面持续用，我建议直接走标准版。标准版的差别不只是功能更多，而是实际用起来更稳，后面返工更少。
```

## 什么时候报加急版

满足以下情况时：

1. 时间窗口非常短
2. 对方明确要优先排期
3. 需求已经相对清楚

典型口径：

```text
如果你这边时间优先，我可以按加急版排期，但前提是样例和规则尽量一次给全，不然加急时间会被反复确认消耗掉。
```

## 报价时一定要顺手说的 3 句话

1. 当前价格默认基于现有样例和规则复杂度
2. 如果文件类型、例外规则、识别难度上升，价格和工期会相应调整
3. 我建议先做一个最小可用版本，再决定是否继续扩展
