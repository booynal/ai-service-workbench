# 成交跟进模板

## 1. 收到需求后的第一条回复

适合在对方刚提交需求、你还没出完整提案时先回：

```text
我已经看过你的需求了，这类项目可以做，而且比较适合先做一个本地可运行的第一版，把核心问题先解决掉。

我这边会先按你提供的资料整理成一个简版方案，里面会包括：
1. 我对需求的理解
2. 建议交付内容
3. 时间预估
4. 三档报价建议
5. 还缺哪些关键信息

如果资料足够，今天就能先给你第一版方案。
```

## 2. 发提案时的配套话术

```text
我先把第一版方案整理好了，你可以先看这几个点：

1. 交付范围是不是对的
2. 哪些约束我理解得不对
3. 你更关注速度、稳定性，还是预算

如果方向没问题，我建议直接按标准版推进，性价比和后续复用会更平衡。
```

## 3. 对方预算低时的回复

```text
如果你现在更关注先低成本验证，这个项目可以先按基础版做，把最核心的流程先跑通。

这样你先确认结果有没有价值，后面如果要加异常处理、日志、更多规则，再往标准版升级。
```

## 4. 对方犹豫时的回复

```text
你可以先不用一次决定完整版本，最稳的方式是先拿样例做第一版验证。

这样你能直接看到：
1. 规则能不能跑通
2. 输出是不是你要的
3. 后续值不值得继续投入
```

## 5. 对方催进度时的回复

```text
如果要加急，我建议把样例、规则、输出格式一次给全，我这边就按加急版排期。

只要输入信息完整，一般当天可以先给你一个初版结果，后面再做微调。
```
