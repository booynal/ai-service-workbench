# 渠道执行矩阵

日期：2026-06-03

## 1. 当前最推荐顺序

按你现在这台 Mac、现有账号和已经完成的落地情况，我建议的渠道顺序是：

1. `GitHub`
2. `微信群 / 朋友圈 / 熟人转介绍`
3. `Fiverr`
4. `Upwork`
5. `Gumroad`

原因不是这些平台谁最大，而是谁最适合你当前已经做好的东西。

## 2. GitHub

### 当前状态

这是你已经真正落地成功的渠道。

你已经有：

1. 公共展示页
2. Issue 收单入口
3. 自动 proposal 生成
4. 自动回帖
5. `proposal-sent` 状态推进
6. 本地 pipeline report

### 适合卖什么

1. 自动化脚本代做
2. 报告整理和分析
3. AI 工作流搭建
4. 开发支持类小项目

### 你还缺什么

1. 真实客户案例
2. 真实成交记录
3. 外部引流动作

### 结论

`GitHub` 是你当前真正可以马上跑起来的公开渠道。

## 3. Fiverr

### 官方规则里最关键的点

1. 创建 Gig 前要先完成 freelancer onboarding。
2. 新卖家最多可有 `4` 个 Gigs。
3. Gigs 和 freelancer account 只能在桌面端创建。
4. 可以设置 `Basic / Standard / Premium` 三档包。
5. Fiverr 要求可修改类服务提供 revision 选项。
6. 价格以 `USD` 设置。

### 适合卖什么

最适合卖“清晰、标准、可包装”的服务：

1. CSV / 文件 / 文档自动化脚本
2. 会议纪要 / 文档整理成报告
3. 轻量 AI workflow setup

### 不建议一开始卖什么

1. 过度开放的长期外包
2. 范围模糊的大项目
3. 需要大量前置沟通的定制系统

### 你还缺什么

1. Fiverr 账号卖家侧 onboarding
2. 至少 3 个标准化 Gigs 标题和封面
3. 英文版案例和英文版报价口径

### 结论

`Fiverr` 是你下一步最值得补的公开平台，因为你现在已经有套餐、案例和报价逻辑，适合直接包装成 Gig。

当前已准备的配套资产：

`Fiverr-Gig-草稿.md`

## 4. Upwork

### 官方规则里最关键的点

1. 账号申请通过后，才能开始投 proposal 或创建 Project。
2. 建议先完成 Getting Started Checklist。
3. Profile 要完整，尤其是技能、作品集、工作经历。
4. Freelancer profile 是最重要的获客资产之一。

### 适合卖什么

`Upwork` 更适合项目型、沟通型、客单价更高一些的服务：

1. 自动化脚本定制
2. 文档和数据处理项目
3. 内部 AI workflow 搭建
4. 开发辅助和技术说明类任务

### 你还缺什么

1. 英文 profile
2. 作品集条目
3. 英文投标模板
4. 对时薪 / 固定价的策略选择

### 结论

`Upwork` 值得做，但比 Fiverr 更依赖 profile 质量和前几单表现。  
如果现在时间有限，优先级应放在 Fiverr 后面。

当前已准备的配套资产：

`Upwork-Profile-和-Proposal-草稿.md`
`Upwork-机会雷达-2026-06-03.md`

## 5. Gumroad

### 官方规则里最关键的点

1. 可以卖数字产品，也可以卖某些服务型产品。
2. 服务型产品对风险更敏感，限制更多。
3. 账号满 `30` 天后，才能提供某些服务类产品。
4. `Commission` 类型是先收 `50%` 定金，完成后再收余下 `50%`。
5. 不能卖“安装或远程代管软件”这类服务。
6. 不能卖长期/线下/超出平台控制范围的服务合同。

### 最适合卖什么

1. 提示词包
2. SOP 模板
3. 资料整理模板
4. 提案模板
5. 一次性交付的数字资产

### 不适合卖什么

1. 远程代安装
2. 长期技术支持
3. 需要持续管理和复杂交付边界的服务

### 你还缺什么

1. 可下载的数字产品
2. 更偏产品化的交付，而不只是接单
3. 英文产品页

### 结论

`Gumroad` 更适合你第二阶段做数字产品，而不是第一阶段拿第一笔服务费。

## 6. 微信 / 熟人转介绍

### 为什么依然值得放前面

1. 不需要额外平台冷启动
2. 成交速度往往比平台快
3. 最容易拿到第一笔真实订单

### 最适合发什么

1. 一条简短服务说明
2. 一个案例链接
3. 一个“把需求发我，我先给方案”的入口

### 你还缺什么

1. 你的收款方式
2. 你愿意公开的朋友圈 / 社群口径

当前已准备的英文外联资产：

`英文外联模板.md`

## 7. 依赖项清单

下面这些不是继续做工作台必须，但会影响你真正收钱：

### 不依赖你补充，也能继续做的

1. 继续完善 GitHub 渠道
2. 补更多样例提案
3. 补更多自动化脚本
4. 准备英文版平台资料

### 需要你补充，才能真正进入成交阶段的

1. 收款方式
2. 你愿意对外公开的联系方式
3. 你最想主打的客户类型
4. 你愿不愿意把现有 GitHub 仓库公开传播
5. 如果要上 Fiverr / Upwork，需要你确认是否直接用这些平台做公开成交

## 8. 当前最现实的渠道结论

如果只看“明天就可能拿到第一笔钱”的概率：

1. `GitHub + 熟人转介绍` 最高
2. `Fiverr` 次高
3. `Upwork` 需要更多 profile 打磨
4. `Gumroad` 更适合第二阶段产品化

一句话总结：

**现在最该做的是继续围绕 GitHub 这条已落地渠道去引流和拿真实线索，而不是立刻分散精力去四个平台同时冷启动。**
