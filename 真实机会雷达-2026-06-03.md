# 真实机会雷达

日期：2026-06-03

## 1. 目的

这份文件只收录比普通 open issue 更接近“真实可能有钱”的机会。

判断标准：

1. 是否出现明确预算信号
2. 是否像真正的交付任务，而不只是维护型 backlog
3. 是否和你现阶段主打服务匹配

## 2. 第一优先观察对象

### Spectral-Finance/lux #71

- 标题：`YouTube Content Creation Pipeline $2,500`
- 链接：`https://github.com/Spectral-Finance/lux/issues/71`
- 信号强度：**高**

为什么值得重点看：

1. 明确出现预算：`$2,500`
2. 是一个清晰的 pipeline 交付需求
3. 既有自动化，也有 workflow 特征

为什么暂时不建议立刻碰：

1. 需求跨度偏大，包含脚本、优化、A/B testing、多语言
2. 如果没有更强的内容生产案例，直接外联命中率不一定高

更适合的用途：

1. 当成高价值市场信号
2. 用来校准你以后如何包装 `workflow setup` 和 `automation pipeline`

## 3. 第二优先观察对象

### ubiquity/business-development #90

- 标题：`GitHub Based Marketing`
- 链接：`https://github.com/ubiquity/business-development/issues/90`
- 信号强度：**中高**

为什么值得看：

1. 明确提到 `Price: 200 USD`
2. 主题本身就是 GitHub 获客和营销
3. 和你现在做的“GitHub 外联 / 候选线索筛选”高度相关

为什么不建议直接碰：

1. 更像一个组织内部增长任务
2. 它本身不一定是对外开放的服务采购

更适合的用途：

1. 当作你现在这套系统的对标参考
2. 反向验证“GitHub 找线索”这条路不是你一个人在想

## 4. 可尝试类对象

### faisalabdullah-commits/csa #2

- 标题：`Need help with workflow setup`
- 链接：`https://github.com/faisalabdullah-commits/csa/issues/2`
- 信号强度：**中**

为什么它依旧是你最现实的试探对象：

1. 虽然没有预算信号，但最像直接服务请求
2. 你的现有能力和素材都已经围绕它准备好了
3. 你现在已经有：
   - 短消息
   - follow-up
   - 价格回复
   - proposal

结论：

**它仍然是最适合你发出第一条陌生试探消息的对象。**

## 5. 不建议优先碰的类型

下面这些类型暂时不该作为第一波：

1. 明显只是内部 roadmap 的 issue
2. 明显是 bugfix / maintenance / triage
3. 需要很强行业背景或大团队协作的长链路项目

## 6. 当前排序

如果按“明天你真的可能做动作”的价值排序：

1. **最适合试探外联：** `csa#2`
2. **最值得当市场信号观察：** `lux#71`
3. **最值得当对标参考：** `ubiquity#90`

## 7. 最短结论

如果你的目标是先迈出真实接触的一步：

1. 发 `csa#2`

如果你的目标是判断未来应该往哪类高客单价项目靠：

1. 观察 `lux#71`
